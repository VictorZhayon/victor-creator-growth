# Tutorial: Send Your First Message with the Anthropic Messages API

The Anthropic Messages API lets you send a prompt to Claude and receive a response. In this tutorial, you'll install the Anthropic Node.js SDK, authenticate with your API key, and make your first successful API call — all in under ten minutes.

---

## Prerequisites

Before you start, make sure you have the following:

- **Node.js v18 or later** installed. Run `node --version` to check.
- An **Anthropic API key**. Get one at the [Anthropic Console](https://console.anthropic.com).
- Basic familiarity with JavaScript and running commands in a terminal.

---

## Step 1: Create a project folder and install the SDK

Open your terminal and run the following commands:

```bash
mkdir anthropic-quickstart
cd anthropic-quickstart
npm init -y
npm install @anthropic-ai/sdk
```

**Expected output:**

```
added 1 package, and audited 2 packages in 1s
found 0 vulnerabilities
```

The SDK is now installed in your project.

---

## Step 2: Store your API key securely

Never hardcode your API key in source files. Instead, store it as an environment variable.

**On macOS/Linux:**

```bash
export ANTHROPIC_API_KEY="your-api-key-here"
```

**On Windows (Command Prompt):**

```cmd
set ANTHROPIC_API_KEY=your-api-key-here
```

To verify it's set, run:

```bash
echo $ANTHROPIC_API_KEY   # macOS/Linux
echo %ANTHROPIC_API_KEY%  # Windows
```

**Expected output:** Your API key printed in the terminal.

---

## Step 3: Write your first API call

Create a file named `index.js` in your project folder and add the following code:

```js
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

const response = await client.messages.create({
  model: "claude-opus-4-5",
  max_tokens: 256,
  messages: [
    {
      role: "user",
      content: "Explain APIs in one sentence, like I'm five years old.",
    },
  ],
});

console.log(response.content[0].text);
```

Because the code uses top-level `await`, add `"type": "module"` to your `package.json`:

```json
{
  "type": "module"
}
```

---

## Step 4: Run the script

```bash
node index.js
```

**Expected output:**

```
An API is like a waiter at a restaurant — you tell it what you want,
it goes to the kitchen (another computer), and brings back exactly what you ordered.
```

The exact wording will vary, but you should see a single clear sentence as Claude's response.

---

## Step 5: Inspect the full response object

Replace the `console.log` line with this to see the complete API response:

```js
console.log(JSON.stringify(response, null, 2));
```

**Expected output (abbreviated):**

```json
{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "An API is like a waiter..."
    }
  ],
  "model": "claude-opus-4-5",
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 21,
    "output_tokens": 36
  }
}
```

**Key fields to understand:**

| Field | Description |
|---|---|
| `content[0].text` | Claude's actual response text. |
| `stop_reason: "end_turn"` | Claude finished naturally and was not cut off. |
| `usage` | Tracks token consumption for billing purposes. |

## Troubleshooting

### `AuthenticationError: invalid x-api-key`

Your API key is missing, incorrect, or not loaded into the environment. Double-check that you exported `ANTHROPIC_API_KEY` in the same terminal session where you run the script. Closing and reopening your terminal clears environment variables — you'll need to export the key again.

### `Cannot use import statement in a module`

Node.js is treating your file as CommonJS instead of ESM. Make sure `"type": "module"` is present in your `package.json`. Alternatively, rename your file to `index.mjs` and run `node index.mjs`.

### `RateLimitError: 429 Too Many Requests`

You've exceeded your API tier's rate limit. Wait 60 seconds and try again. If you hit this repeatedly, review your usage in the Anthropic Console and consider reducing `max_tokens` or spacing out your requests.

## What's next

Now that you've made your first call, you can:

- **Pass a system prompt** by adding a `system` field to your request to give Claude a persona or instructions.
- **Loop messages** to build a multi-turn conversation by appending each assistant response to the `messages` array.
- **Explore the Messages Batches API** to process large volumes of prompts asynchronously at lower cost.

## Bonus: TypeScript fetch() sample

A typed `fetch()` call to the Anthropic `/v1/messages` endpoint, with inline comments written for a beginner.

```ts
// This file makes a typed HTTP request to the Anthropic /v1/messages endpoint.
// It uses the native fetch() API available in modern Node.js (v18+) and browsers.
// No SDK is needed — we're calling the REST API directly.

interface Message {
  role: "user" | "assistant";
  content: string;
}

interface MessagesRequestBody {
  model: string;
  max_tokens: number;
  messages: Message[];
}

interface ContentBlock {
  type: string;
  text: string;
}

interface MessagesResponse {
  content: ContentBlock[];
  stop_reason: string;
}

async function sendMessage(userText: string): Promise<void> {
  const apiKey: string = process.env.ANTHROPIC_API_KEY ?? "";

  if (!apiKey) {
    console.error("Error: ANTHROPIC_API_KEY environment variable is not set.");
    return;
  }

  const body: MessagesRequestBody = {
    model: "claude-opus-4-5",
    max_tokens: 256,
    messages: [{ role: "user", content: userText }],
  };

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": apiKey,
      "anthropic-version": "2023-06-01",
    },
    body: JSON.stringify(body),
  });

  if (!response.ok) {
    const errorText = await response.text();
    console.error(`API error ${response.status}: ${errorText}`);
    return;
  }

  const data = (await response.json()) as MessagesResponse;
  console.log("Claude says:", data.content[0].text);
  console.log("Stop reason:", data.stop_reason);
}

sendMessage("What is one thing a junior developer should know about APIs?");
```
